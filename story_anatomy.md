**STORY ANATOMY**

*A Modular Framework for LitRPG / Progression Fantasy*

v1.0  ·  Working Document  ·  Based on research + personal taste profile

Purpose: This document catalogs every structural layer of a story into discrete, chooseable components with defined options. It is the input layer for a plot generator — not a formula to follow, but a menu to select from. Components at each layer should be compatible with each other and reinforce the central system concept.

**Key principle: The system must have a reason for existing, and that reason should be felt on every page — even if it is not immediately understood.**

# **LAYER 1 — MACRO STRUCTURE**

The bones. Every story needs exactly one choice from each of these components. These decisions constrain everything else.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **1.1  WORLD / SETTING TYPE** |
| **Integration Type** | System apocalypse (Earth gets a system) · Portal fantasy (transported) · Virtual reality (VR/game trapped) · Native world (system always existed) · Hybrid (Earth + portal) | *Determines reader**'**s entry point and what **'**normal**'** looked like before* |
| **World Scale** | Single planet · Multiverse / realms · Dungeon-only · City-state · Galactic | *Sets the ceiling for stakes* |
| **Technology Level** | Pre-modern · Modern real world · Near-future · Sci-fi / post-human · Mixed (magic + tech) |  |
| **1.2  SYSTEM STRUCTURE** |
| **System Visibility** | Full UI (stat screens, notifications) · Partial (some elements visible) · Hidden (felt but not seen) · Contested (protagonist questions it) | *Affects tone — full UI = more gamelike* |
| **System Origin** | Unknown / mysterious · Divine / god-created · Alien intelligence · Human-built gone rogue · Natural evolution of reality · Collapsed civilization remnant | *The mystery to build toward* |
| **System Purpose (hidden)** | Harvest energy / data · Accelerate evolution · Select a champion · Run an experiment · Maintain order · Provide entertainment · Self-preservation · Unknown | *THE backbone. Must be present on every page even before revealed* |
| **System Alignment** | Neutral / indifferent · Actively helpful · Passively hostile · Deceptively benevolent · Openly adversarial · Evolving alongside protagonist |  |
| **System Ceiling** | Hard cap (defined max level/tier) · Soft cap (diminishing returns) · No visible ceiling · Hidden ceiling revealed late | *Defines when the story must end* |
| **1.3  PROTAGONIST ENTRY POINT** |
| **Entry Condition** | Random selection · Chosen / prophesied · Accident / wrong place · Earned / applied · Reincarnation / time reset · Born into it · Forced / trapped |  |
| **Starting State** | Weakest possible (underdog) · Average · Hidden overpowered · Already powerful · Formerly powerful / reset · Disabled / restricted |  |
| **Knowledge Advantage** | None (blank slate) · Future knowledge (time travel) · Meta knowledge (knows it's a game) · Partial insider info · Expert in adjacent field | *Returner**'**s Defiance hook — must be maintained structurally* |
| **1.4  CORE CONFLICT TYPE** |
| **Primary Conflict** | Protagonist vs system · Protagonist vs external threat · Protagonist vs faction/society · Protagonist vs self · Protagonist vs another chosen · Protagonist vs system's creator |  |
| **Conflict Origin** | Imposed by system · Personal vendetta · Ideological · Survival · Protection of others · Curiosity / discovery |  |
| **1.5  TERMINAL ENDING TYPE** |
| **Resolution Mode** | Fulfillment (system achieves goal) · Exposure (true purpose revealed + rejected) · Subversion (protagonist redefines goal) · Replacement (new system established) · Stabilization (world reaches new equilibrium) · Destruction (system ends) | *Must be locked before writing begins* |
| **Protagonist Outcome** | Transcendence (becomes something new) · Return (back to ordinary) · Sacrifice · Apotheosis (becomes part of system) · Rejection (walks away) · Coexistence |  |

# **LAYER 2 — MID STRUCTURE**

The meat. These components shape the texture of the story across its full length. Multiple choices per component are valid.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **2.1  POWER PROGRESSION STYLE** |
| **Progression Mechanic** | Level / XP grinding · Cultivation tiers (named ranks) · Skill tree unlocks · Stat allocation · Class evolution · Bloodline / inheritance · Crafting / creation · Insight / epiphany · Hybrid |  |
| **Progression Pacing** | Rapid early / slow late · Slow and earned throughout · Explosive breakthroughs · Plateau and burst · Continuous steady climb | *Rapid early with meaningful late plateaus tends to feel most satisfying* |
| **Power Ceiling Feel** | Infinite (always more to gain) · Bounded (visible finish line) · Asymptotic (approaches but never reaches) · Tier-gated (hard walls between stages) |  |
| **Power Expression** | Combat-dominant · Crafting / utility · Political / social · Knowledge / information · Creation / building · Hybrid |  |
| **2.2  ANTAGONIST STRUCTURE** |
| **Primary Antagonist Type** | The system itself · A rival chosen · Faction / organization · Ancient/cosmic entity · Former mentor · Protagonist's past self · Corrupt authority · Hidden observer |  |
| **Antagonist Motivation** | Ideological (believes they're right) · Self-preservation · Power accumulation · Revenge · Devotion to system · Fear of protagonist · Competing for same goal | *Best antagonists share the protagonist**'**s goal but via incompatible methods* |
| **Antagonist Reveal Timing** | Known from start · Gradual reveal · Late twist · Multiple antagonists · No single antagonist (systemic) |  |
| **2.3  FACTION / WORLD STRUCTURE** |
| **Faction Landscape** | Single dominant power · Two opposing sides · Multi-faction balance · Hierarchy of powers (tiers/realms) · No factions (individual survival) |  |
| **Protagonist****'****s Faction Role** | Lone wolf · Reluctant member · Hidden agent · Leader / builder · Faction hopper · Creates own faction |  |
| **Political Texture** | Minimal (action-focused) · Moderate (alliances matter) · Heavy (court intrigue) · Systemic (factions ARE the conflict) |  |
| **2.4  STAKES ESCALATION PATTERN** |
| **Escalation Type** | Personal → regional → global → cosmic · Slow burn with single explosion · Multiple escalations with resets · Stakes introduced late · Constant high stakes |  |
| **Cost of Power** | None (pure wish fulfillment) · Physical cost · Moral cost · Relationship cost · Identity cost · All of the above | *Cost creates weight. No cost = hollow victories* |
| **2.5  SYSTEM REVELATION STRUCTURE** |
| **Truth Reveal Pacing** | Drip (one piece per arc) · Two-stage (partial early / full late) · Single late reveal · Protagonist discovers vs told · Red herrings before truth | *The hidden system purpose should leave breadcrumbs from chapter one* |
| **Revelation Trigger** | Protagonist earns it · Antagonist exposes it · Accident / wrong place · System reveals itself · Third party reveals |  |
| **Truth Impact on Protagonist** | Validates their path · Shatters worldview · Forces choice · Changes nothing (they adapt) · Motivates final confrontation |  |

# **LAYER 3 — SUBPLOT STRUCTURE**

The texture. Subplots run parallel to the main plot, intersect it, and complicate it. Each subplot should have its own beginning, middle, and resolution. Pick 2–4 for a single book; more for a series.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **3.1  RELATIONSHIP SUBPLOTS** |
| **Mentor / Teacher** | Genuine guide · Hidden agenda · Flawed teacher · Becomes peer · Betrayal · Dies to motivate · System-assigned | *Most LitRPG uses this — it often becomes formulaic* |
| **Rival** | Same goal, different method · Former ally · System-paired · Parallel power growth · Switches sides · Mirror character | *Best rivals illuminate the protagonist**'**s blind spots* |
| **Companion / Party** | Found family (builds over time) · Assigned team · Temporary allies · Ideologically opposed but cooperating · Rotating cast |  |
| **Romance** | Slow burn · Enemy to ally to lover · Forbidden · Parallel to power growth · Absent (by choice) · Used to show protagonist's humanity | *Optional — only include if it creates plot pressure, not just comfort* |
| **Family / Origin** | Separated and searching · Family as motivation · Family as liability · Chosen family replaces blood · Family secret ties to system |  |
| **3.2  WORLD / SYSTEMIC SUBPLOTS** |
| **Political Intrigue** | Faction war · Power vacuum · Succession crisis · Corruption exposure · Protagonist caught between sides · Building new power base |  |
| **Mystery Thread** | System origin investigation · Who created it · What happened before protagonist · Hidden history of the world · What is the antagonist really doing | *Mystery threads should connect to the hidden system purpose* |
| **World-Building Subplot** | Settlement / city building · Dungeon exploration / mapping · Faction history reveal · Lost civilization discovery · Other chosen / competitors |  |
| **Economic / Resource** | Scarcity driving decisions · Crafting empire building · Trade and political leverage · Information as currency |  |
| **3.3  INTERNAL / CHARACTER SUBPLOTS** |
| **Identity Crisis** | Power changing who protagonist is · Past self vs current self · Moral lines being crossed · What am I becoming · Loss of humanity | *The best LitRPG uses power progression to externalize internal change* |
| **Moral Dilemma Arc** | Ends vs means conflict · Collateral damage · Choosing between people they care about · System rewards evil · Protagonist becomes the thing they fought |  |
| **Secret / Hidden Truth** | Protagonist hiding something · Past they're running from · Ability they conceal · True identity · Knowledge they can't share |  |
| **Obsession / Fixation** | Revenge driving irrational decisions · Power at any cost · Protecting one person above all · Completing a task the system set |  |
| **3.4  MYSTERY / THRILLER SUBPLOTS** |
| **Investigation Thread** | Who is the real antagonist · What happened to a missing person · What does the system actually want · Who sent the attack |  |
| **Conspiracy** | Faction hidden within a faction · System has a secret controller · Multiple chosen working at cross purposes · The tutorial was a trap |  |
| **Countdown / Race** | Limited time before catastrophe · Other chosen gaining too fast · System event approaching · Resource about to run out | *Creates urgency without requiring constant combat* |

# **LAYER 4 — PROTAGONIST ARCHETYPE**

The lens. Who the protagonist is shapes every choice, relationship, and conflict. These are not personality types — they are structural roles the protagonist plays in relation to the system.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **4.1  RELATIONSHIP TO THE SYSTEM** |
| **The Exploiter** | Treats the system as a tool to be gamed. Finds loopholes, optimizes ruthlessly, doesn't care about the system's purpose — only what it provides. | *DCC**'**s Carl. Risk: can feel amoral if not grounded* |
| **The Investigator** | Driven to understand why the system exists. Power is a means to the truth, not the goal. | *Mother of Learning**'**s Zorian. Best for mystery-heavy plots* |
| **The Defier** | Resists or subverts the system's intent. May play along but works against it structurally. | *Defiance of the Fall**'**s Zac. Works when system is oppressive* |
| **The True Believer** | Initially aligned with system's purpose. Arc involves discovering the truth changes everything. | *Interesting when the system turns out to be wrong* |
| **The Returner** | Has foreknowledge. Arc is about how knowledge without wisdom is still dangerous. | *Returner**'**s Defiance. Must maintain knowledge as structural pressure* |
| **The Builder** | Uses power to construct — cities, armies, organizations. System is infrastructure. | *Ten Realms. Satisfying when building has stakes* |
| **The Accidental** | Dragged in unwillingly. Arc is accepting the new reality and finding agency within it. | *HWFWM**'**s Jason. Works when fish-out-of-water is played for texture not comedy* |
| **4.2  CORE FLAW (required)** |
| **Flaw Type** | Arrogance · Isolation · Recklessness · Obsession · Trust issues · Moral rigidity · Fear of intimacy · Overprotectiveness · Nihilism | *The flaw should be tested by the system**'**s pressure, not ignored by it* |
| **Flaw Arc** | Overcomes it · Learns to manage it · It costs them something major · It becomes a strength in context · Never fully resolves (realistic) |  |
| **4.3  PROTAGONIST COMPETENCY STYLE** |
| **How They Win** | Outfights · Outthinks · Outlasts · Out-networks (allies) · Out-crafts · Gets lucky consistently · Breaks the rules |  |
| **Competency Reveal Pacing** | Shown immediately · Hidden and revealed dramatically · Grows visibly over time · Underestimated by everyone · Overestimated by protagonist |  |

# **LAYER 5 — BEAT STRUCTURE**

The skeleton. Synthesized from Save the Cat (15 beats), Hero's Journey (Campbell), and Seven-Point Structure (Dan Wells), adapted for LitRPG's multi-book serialized format. A single book needs all 15 beats. A series distributes them across volumes with micro-versions of each beat per book.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **5.1  ACT ONE — SETUP (0–25%)** |
| **Opening Image** | Snapshot of protagonist's world BEFORE the system. Shows their flaw, their ordinary, their unfulfilled need. | *The **'**before**'** photo. Should mirror the final image* |
| **Theme Stated** | Someone (often not protagonist) says something that hints at the story's real question. Protagonist doesn't understand it yet. | *In LitRPG: often a statement about what power costs* |
| **Setup / World Introduction** | Establish the world, the rules, who the protagonist is, what they want, what they fear. |  |
| **Catalyst / Inciting Incident** | The system arrives, activates, or reveals itself. The protagonist's ordinary world is shattered. No going back. | *In system apocalypse: the integration. In VR: the login that doesn**'**t log out* |
| **Debate** | Protagonist resists, questions, or processes the new reality. Do they engage or retreat? | *Often skipped in LitRPG — leads to protagonist feeling reactive not active* |
| **Break into Act Two** | Protagonist commits. Enters the system willingly or is forced fully in. The real story begins. |  |
| **5.2  ACT TWO A — FUN ****&**** GAMES (25–50%)** |
| **B Story Introduction** | The subplot begins — usually a relationship that will carry the theme. | *In LitRPG: the party forms, the rival appears, the mentor is introduced* |
| **Fun and Games** | The promise of the premise. This is the reader's reason for picking up the book. Protagonist explores their new power. World opens up. | *Dungeon crawls, skill unlocks, early victories, world discovery. This section should deliver what the cover promises* |
| **Midpoint** | A false victory or false defeat. Stakes double. Protagonist either achieves their surface goal (and discovers it wasn't enough) or suffers a major public setback. | *In multi-book: midpoint of book 1 is often end of book 1* |
| **5.3  ACT TWO B — BAD GUYS CLOSE IN (50–75%)** |
| **Bad Guys Close In** | External pressure mounts. Internal cohesion fractures. The protagonist's flaw actively creates problems. The antagonist gains. | *In LitRPG: power ceiling hit, allies lost, system reveals something alarming* |
| **All Is Lost** | The protagonist's worst moment. A loss that feels permanent. Something dies (literally or figuratively). | *Must earn the finale. DCC does this brutally — it hits because you believe it* |
| **Dark Night of the Soul** | Protagonist alone with their failure. Must confront what they did wrong, who they actually are, what they really want. | *In LitRPG: often rushed. Give it space. This is where theme crystallizes* |
| **5.4  ACT THREE — RESOLUTION (75–100%)** |
| **Break into Act Three** | New idea, new angle, new understanding of the theme. Protagonist finds a way forward that uses everything they've learned. | *In LitRPG: the realization that the system**'**s purpose changes their approach* |
| **Finale** | Protagonist faces final challenge using new self. Subplots resolve. Antagonist confronted. System's purpose revealed or resolved. | *Should feel inevitable in retrospect. Every earlier beat pointed here* |
| **Final Image** | Mirror of Opening Image — shows how much has changed. The 'after' photo. | *In LitRPG: what does their stat screen look like now? What do they value?* |

# **LAYER 6 — MICRO TEXTURE**

The seasoning. These choices shape feel and voice without changing the structure. Pick as many as apply.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **6.1  TONE** |
| **Primary Tone** | Grim / serious · Comedic · Cerebral / puzzle-focused · Action-dominant · Horror undertone · Hopeful despite darkness · Cynical |  |
| **Tone Consistency** | Constant throughout · Shifts per arc · Contrast used deliberately (comedy before tragedy) · Evolves with protagonist |  |
| **6.2  PACING RHYTHM** |
| **Chapter Structure** | Single POV · Multiple POV alternating · Dual timeline · Retrospective narration · Real-time only |  |
| **Action / Rest Ratio** | Constant action · Action-recovery cycles · Long quiet arcs punctuated by violence · Mostly tension with rare action | *Most serialized LitRPG skips rest — this creates reader fatigue* |
| **Information Release** | Front-loaded (reader knows more than protagonist) · Drip-fed (discover together) · Delayed (protagonist knows, reader doesn't) · Mystery box (neither knows) |  |
| **6.3  RECURRING MOTIFS** |
| **Thematic Motif** | A recurring image, object, phrase, or situation that carries the theme | *E.g. DCC: the loot box (hope/reward as manipulation). Should appear 3+ times in different contexts* |
| **System Motif** | A recurring system mechanic that evolves in meaning as the story progresses | *E.g. a skill that starts as survival tool, becomes ethical problem, ends as theme statement* |
| **6.4  CHAPTER HOOKS** |
| **Hook Type** | Cliffhanger (action unresolved) · Revelation (new information) · Emotional (relationship beat) · Question (mystery introduced) · Callback (earlier beat pays off) · Irony (character acts on wrong assumption) | *Vary hook types — same hook every chapter creates numbness* |
| **6.5  HUMOR STYLE (if applicable)** |
| **Humor Type** | Protagonist's internal monologue · Fish-out-of-water misunderstanding · Absurdist system notifications · Deadpan companion · Situational irony · Self-aware genre commentary | *DCC uses all of these. Most LitRPG uses only one and overdoes it* |

# **LAYER 7 — SERIES ARCHITECTURE**

For stories planned across multiple books. Each book needs its own mini-arc with a satisfying resolution, while the larger system purpose unfolds across the series.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **7.1  SERIES STRUCTURE TYPE** |
| **Arc Type** | Each book = self-contained story · Each book = one phase of larger arc · Continuous narrative (no book-level resolution) · Anthology (same world, different characters) | *Continuous narrative is the LitRPG default — and why most series drift* |
| **Series Ceiling** | Defined (X books total) · Open-ended with defined ending · Open-ended with thematic endpoint · No planned ending | *Define this before writing book one* |
| **7.2  REVELATION DISTRIBUTION** |
| **System Truth Reveal Schedule** | One major revelation per book · Slow drip across all books · One false reveal then real reveal · All revealed in final book | *Map the breadcrumbs before writing. Each breadcrumb in book 1 should pay off* |
| **Protagonist Power Ceiling Per Book** | Clear tier achieved per book · Continuous growth with no per-book milestone · Power plateau per book resolved in next |  |
| **7.3  ANTI-DRIFT MECHANISMS** |
| **Structural Lock** | Hard system ceiling defined · Non-system stabilizing goal that exists independently · Thematic question that must be answered · External deadline (event, threat, timer) | *Pick at least two. Drift happens when none of these exist* |
| **Series Bible Elements** | World rules that cannot be violated · System rules that cannot be retconned · Character decisions that are permanent · Stakes that cannot be reset | *Write these down before book one. Commit publicly if possible* |

# **LAYER 8 — THEMATIC ANCHORS**

The soul. Every structural component should ultimately serve the theme. Pick one primary theme and one secondary. The system's hidden purpose should be a manifestation of the theme.

| **COMPONENT** | **OPTIONS / VARIANTS** | **DESIGN NOTE** |
| --- | --- | --- |
| **8.1  PRIMARY THEME OPTIONS** |
| **Power vs Humanity** | Does gaining power make you less human? What do you sacrifice to become stronger? At what point is the cost too high? | *Most common in LitRPG — often unexplored because power gain is framed as pure positive* |
| **Control vs Freedom** | Who controls the system? Who benefits? Is order worth the cost of autonomy? | *Perfect for system-as-antagonist or system-as-benevolent-oppressor stories* |
| **Survival vs Morality** | What are you willing to do to survive? Does survival without values constitute living? | *Works best in apocalypse settings where the system removes normal social rules* |
| **Progress vs Identity** | Does growing mean leaving behind who you were? Is transformation loss or gain? | *Strong for cultivation / tier-advancement stories* |
| **Knowledge vs Consequence** | Is knowing everything a gift or a burden? Does foreknowledge destroy agency? | *Ideal for time-travel, investigator, or simulation-type systems* |
| **Individual vs Collective** | Does the protagonist's growth serve only them or the world? What do they owe others? | *Strong for builder / nation-building archetypes* |
| **Truth vs Comfort** | Is understanding the system's real purpose worth shattering the protagonist's worldview? | *Best when the system reveal is the climax* |
| **8.2  THEMATIC QUESTION FORMAT** |
| **Structure** | Frame your theme as a YES/NO question the story answers. The answer should not be obvious at the start. E.g.: 'Can you build something worth protecting without becoming the thing that threatens it?' | *The ending is your answer. Work backward from there* |

# **COMPATIBILITY RULES**

Not all combinations work. These are guardrails — component choices that tend to create internal contradictions if combined naively.

- If System Purpose = Entertainment → Protagonist cannot be fully autonomous. The audience's wants must constrain every decision.

- If Protagonist Archetype = Returner (foreknowledge) → The foreknowledge MUST create compounding problems. If it stops mattering by book 2, the premise is broken.

- If System Alignment = Neutral → The system's purpose must still create friction. Neutral ≠ inert. The system wants something even if it doesn't care about the protagonist.

- If Theme = Knowledge vs Consequence → The mystery of system origin must be the engine, not a subplot. Solving it IS the ending.

- If Series Architecture = Open-ended → You MUST have at least two anti-drift mechanisms locked. No series ceiling + no thematic question = drift by book 3.

- If Cost of Power = None → The protagonist must have a relationship or identity subplot that creates stakes. Costless power needs human cost instead.

- If Ending Type = Exposure → Plant the first breadcrumb on page one. If the truth can be revealed without any prior setup, it's a twist, not a resolution.
